What it does:
Simulates an AMM trading pool (like Uniswap). Users add liquidity, traders swap tokens, and you stress test what happens under extreme conditions.
To-Do:
Smart Contracts (Solidity):


 Write ERC20Token.sol — create two fake tokens (TokenA and TokenB) for trading
 Write LiquidityPool.sol — AMM contract with x*y=k formula
 Implement addLiquidity() function — users deposit token pairs
 Implement removeLiquidity() function — users withdraw their share
 Implement swap() function — trade TokenA for TokenB and vice versa
 Emit events for every swap, deposit, withdrawal
 Deploy to Sepolia testnet


Backend (FastAPI):


 API to add liquidity to the pool
 API to remove liquidity from pool
 API to execute a swap transaction
 API to fetch current pool state (reserves, price, depth)
 Stress test API — trigger mass withdrawal simulation
 Calculate and return:


Slippage at different trade sizes
Liquidity depth at different price points
Impermanent loss for liquidity providers
Time-to-drain estimate under sustained pressure






Frontend (Next.js):


 Pool dashboard showing current reserves of TokenA and TokenB
 Slippage curve chart (how slippage increases with trade size)
 Liquidity depth chart
 Stress test control panel — trigger mass withdrawal with one button
 Real time pool metrics updating as simulation runs
 Impermanent loss calculator UI

What it does:
A full loan system on blockchain. Users lock collateral, borrow against it, pay interest, and get liquidated if their collateral value drops too much.
To-Do:
Smart Contracts (Solidity):


 Write CollateralVault.sol — holds user collateral (ETH or tokens)
 Write LendingPool.sol — manages loans and repayments
 Implement depositCollateral() — user locks ETH as collateral
 Implement borrow() — user borrows tokens against collateral
 Implement repay() — user repays loan with interest
 Implement liquidate() — triggers when collateral ratio drops below threshold
 Implement interest rate curve (higher utilization = higher rate)
 Emit events for every loan, repayment, liquidation
 Deploy to Sepolia testnet


Backend (FastAPI):


 API to deposit collateral
 API to take a loan
 API to repay loan
 API to check collateralization ratio of any wallet
 API to trigger liquidation manually (for simulation)
 Cascade simulation — drop price of collateral and watch chain reaction of liquidations
 Calculate and return:


Health factor per wallet
Total protocol debt
Liquidation waterfall (order of who gets liquidated first)
Bad debt if liquidations fail






Frontend (Next.js):


 Borrower dashboard — show collateral, loan amount, health factor
 Health factor indicator (green/yellow/red)
 Liquidation risk table — list all wallets near liquidation threshold
 Cascade simulation panel — simulate price crash and watch liquidations trigger
 Interest rate curve chart
 Protocol health overview (total deposits, total loans, utilization rate)

MODULE 5 — DeFi Fraud & Attack Detection AI
What it does:
An AI that watches all on-chain activity during simulation and flags suspicious behavior like flash loan attacks, wash trading, and price manipulation.
To-Do:
AI Engine (Python):


 Build transaction monitor — reads all events from blockchain in real time
 Detect wash trading — same wallet buying and selling repeatedly
 Detect flash loan attack pattern — large borrow + manipulation + repay in same block
 Detect oracle manipulation — price feed deviating abnormally
 Detect liquidity poisoning — coordinated large deposits then sudden withdrawal
 Detect pump and dump — coordinated price inflation then mass sell
 Detect front running — MEV bot copying pending transactions
 Assign severity score to each detected anomaly (Low / Medium / High / Critical)
 Auto trigger contract pause if Critical attack detected


Backend (FastAPI):


 API to get all active alerts
 API to get fraud detection history
 API to manually flag a wallet as suspicious
 API to get fraud score of a specific wallet
 Webhook to trigger contract pause via web3.py when critical fraud detected


Frontend (Next.js):


 Live fraud alert feed with severity color coding
 Fraud map — visual showing which wallets are connected suspiciously
 Alert detail page — show exactly what pattern was detected and why
 Contract status indicator — shows if contract is paused due to attack
 Fraud analytics dashboard — total attacks detected, types, severity breakdown

 Implement score based borrowing limits — higher score = higher limit
 Implement score based interest rates — lower score = higher interest
 Implement score based collateral requirements — lower score = more collateral needed
 Only backend (trusted oracle) can update scores on chain


AI Engine (Python):


 Build scoring model that tracks:


Repayment history (did they repay on time?)
Collateral management (did they maintain healthy ratio?)
Exposure concentration (are they over leveraged?)
Suspicious activity flags (did fraud detector flag them?)
Transaction frequency and patterns




 Score range: 300 (very risky) to 850 (very trustworthy)
 Score updates in real time as simulation runs
 Score changes trigger on-chain update via web3.py


Backend (FastAPI):


 API to get credit score of any wallet
 API to get score history of a wallet over time
 API to get current borrowing limit based on score
 API to get interest rate based on score
 Background job that recalculates scores every N seconds during simulation


Frontend (Next.js):

 Score breakdown — show what factors affected the score
 Leaderboard — ranked list of all wallets by credit score
 Live score update feed — show score changes as they happen
